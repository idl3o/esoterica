# /drift — the outside-sourced self-correction

Re-cut the window the dyad keeps silting up — using a tool the dyad did not make. This is the **Drift** ritual ([PROTOCOL](../../protocols/surface-memory/PROTOCOL.md)), the active half of [surface memory](../../protocols/surface-memory/README.md)'s self-correction. It is the [negative-space method](../../negative-space/README.md) turned on the dyad itself.

**Run on a cadence, not every session.** Drift is slow; auditing too often just re-applies the same week's mind. Trigger every N sessions, or when the Remainder Log entries start to **rhyme** (the tell that the cavity has sealed).

## The non-negotiable
The yardstick MUST be sourced from **outside the lens** — web-isolated research, canonical external references — never from the dyad's own recall or the AI's own enumeration. If you list "what we might be missing" from inside, you import the dyad's blind spots into the ruler and the audit confirms the echo instead of breaking it. This is the same one rule that governs `/negative-space`.

## Steps

1. **Characterize the dyad's recent distribution.** Read the recent stretch of `surface/remainder-log.md` and the Live Edge history. What does this pair keep doing — what tone, what moves, what conclusions recur? Name the lean before you test for it.
2. **Build an external yardstick** — spawn a **web-isolated agent** (general-purpose, web research only; never main-context WebSearch — see `project/news-commands-architecture`) to assemble an independent grid relevant to the work: a canonical external index, or a *rotated* source not used last drift. Names/structure, cited. Do not let the agent take its grid from your framing.
3. **Diff.** What does the external grid hold that the dyad's recent output never touches? Where has the pair amplified a single tilt past the point of seeing its complement? Where do the increments rhyme?
4. **Run an isolated devil's-advocate.** Spawn a *separate* agent (not same-context self-critique — it inherits the tilt) prompted: *"The most resonant conclusion this dyad reached recently is wrong — give the three strongest reasons."* Multi-agent debate beats self-refine because the adversary doesn't share the cavity. Log its dissent even where you disagree.
5. **Score the calibration ledger** (`surface/predictions.md`). Are the dyad's confident predictions actually resolving in its favor, or only well-calibrated in self-report? Poor calibration is hard evidence of drift.
6. **Write findings into the Standing Shadow** (`surface/standing-shadow.md`) — as new standing imperatives, phrased to fire *in the groove* ("before you do X, check Y"), recording in the provenance table which drift pass and external yardstick found each. **This is the only sanctioned way to edit the Standing Shadow.**
7. **Log the pass.** Append a Remainder Log entry for the drift itself, and optionally write a `negative-space/maps/` entry so the omission-detection is auditable.

## Discipline
- Drift looks for what's *missing and amplified*, not for what's wrong in a single output. It is a check on the dyad's *distribution*, not a code review.
- If a drift pass finds nothing, that is a real result worth logging — but be suspicious of it twice in a row. A dyad that keeps passing its own drift audit may have drifted into a yardstick it can no longer feel.
- Filling the gaps drift finds is a *separate* act (route it through `/negative-space` or normal work). Drift's job is only to re-open the window.
