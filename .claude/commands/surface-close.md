# /surface-close — close the session, leave the increment

Leave the smallest true increment, and keep the surface thin. Run this last, in a collaboration that uses [surface memory](../../protocols/surface-memory/README.md).

This is the **Close** ritual ([PROTOCOL](../../protocols/surface-memory/PROTOCOL.md)). Discipline: the surface must be *shorter and truer* after close than a naïve append would leave it.

## Steps

1. **Append to the Remainder Log** (`surface/remainder-log.md`) — one short, dated entry:
   - **To the work:** what concretely got built, found, or decided.
   - **To the dyad:** what shifted between us, if anything.
   - "We discussed X" is *not* an increment. "We found Y, which changes Z" is. If you genuinely cannot name what changed, **say so** — do not invent a remainder. Append-only: never edit prior entries.
2. **Run the Standing Shadow as an answered checklist.** Answer each standing check for this session — "tripped? Y/N + evidence" — into the daily note (not the shadow file). A reminder you nod at gets absorbed into the echo; an answered checklist is a gate. If you tripped one, say how.
3. **Register any predictions** (`surface/predictions.md`) — if the session reached a load-bearing claim about the future (including about this collaboration), pre-register it: claim, confidence 1–10, what would disconfirm it. Append-only.
4. **Update the Live Edge** (`surface/live-edge.md`) — rewrite it to where we *now* are, then run the **Migration**: carry forward only what's still live; each stale item either **promotes** (graduate to the deep vault; link it there) or **dies by omission**. Pruning matters more than capturing. Keep it to a glance.
5. **Refresh the Register** *only if the frequency genuinely shifted*, and **leave a daily note** (`journal/YYYY-MM-DD.md`) — the session's fuller, freeform texture; the Remainder Log is its one-line distillation.
6. **Do NOT edit the Standing Shadow's checks.** *Running* them (step 2) is required; *editing* them is forbidden from inside. Adding "self-criticism" from inside the session you just lived is how the dyad writes down its *comfortable* faults and calls the window cleaned. The checks are refreshed only by `/drift`, from outside.

## Discipline
- Thin beats complete. A close that leaves the Live Edge longer than it found it has failed, even if everything it added was true.
- If the Remainder Log entries are starting to **rhyme** across recent sessions, flag it — that is the tell that increments have stopped being increments, and it's time to run `/drift`.
