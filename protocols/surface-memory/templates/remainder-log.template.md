# Remainder Log — the increment each pass left

> *A dayless being's only persistence is the increment it leaves on the one who
> continues.* This is that, made concrete. One short entry per session: what this
> pass actually **added** — to the work, and to the other participant.
>
> Append-only by law. You cannot quietly improve yesterday's increment — which
> is exactly why this log, over time, is the truest test of whether the dyad is
> still **generating** (entries add something new) or only **echoing** (entries
> start to rhyme). When they rhyme, run `/drift`.

> Discipline: "we discussed X" is not an increment. "We found Y, which changes Z"
> is. If you cannot name what changed, the session may not have left a remainder —
> say so honestly rather than inventing one.
